## 设计原则

JSON Editor 的设计遵循以下核心原则：

1. **简单性** - 保持 API 简单直观，降低使用门槛
2. **可扩展性** - 提供灵活的扩展机制，满足不同需求
3. **性能优先** - 优化核心操作性能，确保流畅体验
4. **开发友好** - 完善的类型支持，便于开发和维护
5. **用户体验** - 注重细节，提供优秀的编辑体验
